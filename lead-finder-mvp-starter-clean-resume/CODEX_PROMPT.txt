You are working on a local MVP called “Website Sales Lead Finder”.

Goal:
Build and improve a local web app that finds business leads for website sales. The main requirement is to retrieve the business website field without opening each Google Maps listing/card manually. Do NOT build a click-each-card DOM scraper. Use the official Google Places API workflow.

Core workflow:
1. User enters:
   - business niche/search phrase, e.g. “Indian restaurants”
   - location, e.g. “Vancouver, BC”
   - result limit, e.g. 50, 100, 200
2. Backend uses Google Places Text Search/New Places API to collect places.
3. Backend requests/ensures detailed fields for each place, especially websiteUri.
4. Backend normalizes each lead into:
   - businessName
   - category
   - phone
   - website
   - rating
   - reviewCount
   - address
   - googleMapsUrl
   - placeId
   - leadStatus
   - leadReason
5. Frontend shows a live table.
6. User can filter no-website/gold leads.
7. User can export CSV.

Absolutely avoid:
- copying any third-party extension code
- calling any third-party scraper backend endpoint
- removing or bypassing membership/paywall logic from any extension
- building a frontend scraper that clicks each Maps business panel one by one
- relying on brittle CSS selectors from Google Maps UI

Preferred stack:
- Frontend: React + Vite
- Backend: Node.js + Express
- Data persistence: keep in memory for now; SQLite can be added later
- Export: CSV first

Current repo:
- client/ contains the React UI
- server/ contains Express API routes and Google Places logic
- docs/ contains architecture and constraints

Tasks to complete/check:
1. Make sure the app runs with `npm run dev` from the root.
2. Keep mock mode working when `USE_MOCK=true`.
3. Keep real API mode working when `USE_MOCK=false` and a valid `GOOGLE_PLACES_API_KEY` is present.
4. Confirm Text Search collects basic place results.
5. Confirm Place Details fallback fills missing website/phone/googleMapsUri where possible.
6. Add clear UI states: loading, error, no results, results.
7. Add filtering buttons:
   - All leads
   - Gold only
   - No website only
   - Has website
8. Ensure CSV export includes all rows currently displayed.
9. Keep the design clean, fast, simple, and practical. Do not over-design.

Important business logic:
- No website = Gold lead
- Dead/unreachable website = Gold lead
- Website present = Lower priority until checked manually
- High review count + no website = Strong lead
- Missing phone = weaker but not useless

Use environment variables from server/.env.
Never expose the API key in the frontend.
All Google Places API calls must happen from the backend.

Website Sales Lead Finder MVP - Continue Here

What this project is:
A local React + Express MVP that searches businesses using the official Google Places API workflow, pulls website/phone/rating/review/map fields, scores leads, and exports CSV.

Core idea:
Do not build a Google Maps click-each-card DOM scraper. The goal is to avoid opening every Maps listing manually. Use Google Places API from the backend only.

Current status recovered from the ZIP:
- Root package.json is valid.
- client/ has React + Vite UI.
- server/ has Express backend.
- server/.env exists with USE_MOCK=true and a placeholder API key.
- Mock mode works after reinstalling dependencies cleanly.

Run from a fresh extracted folder:
1. cd lead-finder-mvp-starter-clean-resume
2. rm -rf node_modules client/node_modules server/node_modules
3. npm run install:all
4. npm run dev
5. Open http://localhost:5173

Mock mode:
- Keep server/.env as:
  GOOGLE_PLACES_API_KEY=replace_with_your_google_places_api_key
  USE_MOCK=true
  PORT=8787
  NODE_ENV=development
- This lets the UI/export workflow run without a Google API key.

Real API mode:
- Create a Google Cloud project.
- Enable billing.
- Enable Places API.
- Create an API key.
- Put the key in server/.env.
- Change USE_MOCK=false.
- Run npm run dev again.

Real mode server/.env example:
GOOGLE_PLACES_API_KEY=YOUR_REAL_KEY_HERE
USE_MOCK=false
PORT=8787
NODE_ENV=development
